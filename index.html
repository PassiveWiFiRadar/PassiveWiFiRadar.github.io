PROJECT PROPOSAL

Motivation



Computer vision techniques are not limited to processing camera data, nor is camera deployment always feasible in areas where privacy is of concern, or when opaque objects may block views. It is implied but not often discussed in computer vision courses that image processing techniques may be applied to any 2D signal representation, including spectrograms from wireless signals. By representing signals as images, we can process them with CNNs and similar computer vision techniques similarly to images from a camera. The field of wireless sensing is a promising approach to detecting various features about the environment in situations where cameras may not be appropriate, such as in private homes.

Problem Statement



The goal of this project is to explore using WiFi as a unique form of computer vision by exploiting the way the physical environment interacts with wireless signals. Specifically, we intend to use Doppler spectrograms from a passive WiFi radar (PWR) dataset for human activity recognition (HAR) by processing the data as images.

Approach and Experiments

We plan to train deep learning models to perform visual classification of various human activities based on Doppler spectrograms, such as walking, running, standing, and sitting. The dataset we intend to use is a subset of the OperaNET dataset [1]. This data is presented in .mat files which we will load into Google CoLab for preprocessing. Since three passive receivers were used in the data collection process, we effectively have three-channel images to work with.

We hope to reproduce the PWR-related subset of findings from the related research paper, as well as to demonstrate that computer vision tools can generalize to non-camera images. Another team aims to work in parallel with us by using WiFi channel state information (CSI) from the same dataset, which will allow us to compare results in a complementary way while having distinct deliverables. We plan to use open-source libraries such as OpenCV, Tensorflow, and Pytorch. To verify the performance of our model, we intend to use industry-standard metrics such as confusion matrices and F1 score.
